Class Overview
==============
The WishList class represents a user's collection of desired products. It maintains a reference to the owner (Utente) and a list of products (ArrayList<Prodotto>).
Key responsibilities include:
- Managing the association with a user.
- Managing the list of products.
- Enforcing non-null constraints on the user and the product list during valid state transitions (constructors and setters).

Key State Variables:
- utente (Utente): The owner of the wishlist. Can be null if the default constructor is used, but setters and parameterized constructors enforce non-nullity.
- prodotti (ArrayList<Prodotto>): The list of items. Initialized to empty in constructors, enforced non-null in setter.

Constructors and Methods Analysis
=================================

1. WishList()
-------------
Default constructor. Initializes the product list to an empty ArrayList.
Parameters: None
State: Clean initialization.

Categories and Choices:
- Initialization:
  - Default: Creates object with null utente and empty product list.

Test Frames:
- F1: Default initialization.
  - Setup: Call new WishList().
  - Expected: utente is null, products list is not null and empty.

Suggested JUnit 5 Test:
- testDefaultConstructor
  - Arrange: None.
  - Act: WishList wl = new WishList();
  - Assert: assertNull(wl.getUtente()); assertNotNull(wl.getProdotti()); assertTrue(wl.getProdotti().isEmpty());

2. WishList(Utente utente)
--------------------------

Parameters: utente (Utente)

Categories and Choices:
- utente:
  - Valid: A valid Utente object.
  - Invalid: null.

Test Frames:
- F2: Valid utente.
