SYNTAX AND STYLE GUIDE
================================================================================

[FIELDS]
- Mark private fields as: /*@ spec_public @*/
- If the field is an Object and can be null: /*@ spec_public nullable @*/
- If the field is a primitive (int, double), NEVER use nullable.
- Define invariants for fields that must adhere to specific rules (e.g., invariant id >= 0;).

[PURE METHODS]
- Getters and query methods must be marked with the "pure" keyword inside the JML block.
- Example: 
  /*@ 
    @ ensures \result == field;
    @ pure
    @*/

[CONSTRUCTORS]
- Ensure the JML reflects the actual state after construction (even if fields are null).

[METHODS WITH EXCEPTIONS]
- Use the following template strictly:

  /*@
    @ public normal_behavior
    @   requires argument != null;

    @
    @ also
    @
    @ public exceptional_behavior
    @   requires argument == null;
    @   assignable \nothing;
    @   signals (IllegalArgumentException e) true;
    @*/
